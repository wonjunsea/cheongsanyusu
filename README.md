## 서비스명: 청산유수

## 서비스 소개

**청산유수**는 투자 직전, 빚투·영끌 청년들의 계좌 청산을 막고 고위험 금융 상품의 위험성을 인지시키는 **30초 교육 솔루션**입니다.

청년 투자자가 레버리지, 인버스, 급등주 등 고위험 상품에 진입하기 전 짧은 시뮬레이션과 게임을 통해 상품의 위험성을 직접 체험할 수 있도록 설계했습니다.

복잡한 금융 교육을 길게 듣는 방식이 아니라, 실제 투자 의사결정 직전에 초보자도 쉽게 이해할 수 있는 체험형 콘텐츠를 제공하여 무리한 레버리지 진입을 줄이고 청년 투자자를 보호하는 것을 목표로 합니다.

---

## 해결하고자 하는 문제

| 문제 | 설명 |
|---|---|
| 청년 빚투·영끌 증가 | 소득이 불안정한 청년층이 위험을 충분히 이해하지 못한 채 고위험 상품에 진입하는 문제가 증가하고 있습니다. |
| 고위험 상품 이해 부족 | 레버리지, 인버스, 급등주 등은 손실 구조가 복잡하지만 투자자는 수익 가능성에만 집중하기 쉽습니다. |
| 기존 교육의 낮은 접근성 | 기존 금융 교육은 수동적이고 길어 실제 투자 직전의 사용자를 붙잡기 어렵습니다. |
| 투자 직전 개입 부족 | 위험한 선택이 일어나는 바로 그 순간에 위험을 체감시키는 장치가 부족합니다. |

## 주요 기능

| 기능 | 설명 |
|---|---|
| 진입 훅 화면 | 고위험 상품(레버리지·인버스·급등주) 클릭 시, 채팅방이 열리기 전에 케이스별 한 줄 후킹 문구("2배 ETF면 더 벌어야 하는 거 아닌가요?" 등)를 전체 화면으로 먼저 보여줘 주의를 환기합니다. |
| AI 금융 멘토 롤플레잉 | 자유 대화가 아니라 **객관식 4턴(질문 3회 + 최종 1회)**으로 진행됩니다. 매 턴 LLM이 사용자의 이해 방식(mental_model)·위험 인식(risk_belief)·시장 해석 프레임(market_frame)을 함께 추론해 다음 질문의 방향을 정합니다. |
| 케이스별 맞춤 시나리오 | 클릭한 종목에 따라 고정된 케이스로 진입합니다 — 에코비트(급등주) → FOMO/추격매수, KODEX 레버리지 → 음의 복리(변동성 손실), 200선물인버스2X → 인버스 ETF의 하락 베팅 착각. 케이스마다 자주 나타나는 착각 패턴과 질문 전략이 다르게 설계되어 있습니다. |
| 체험 콘텐츠 연결 | 대화의 최종 턴에서 나온 행동 선택지를 누르면, 케이스에 맞는 체험 화면(급등주 호가창 게임, 음의 복리 시뮬레이션, ETF 체험(준비 중))으로 자동 연결됩니다. |
| 대화 기록 저장 | 사용자(브라우저별 랜덤 UUID)와 케이스 단위로 매 턴의 질문·선택 답변을 SQLite에 적재합니다. |
| 체험 허브 · 용어 퀴즈 · 모의투자 | 랜딩에서 별도로 접근 가능한 보조 콘텐츠 — 슬라이더로 조작하는 음의 복리 체험, 5문항 금융 용어 퀴즈, 모의투자 시뮬레이션을 제공합니다. |

## 사용 기술

| 분류 | 기술 | 역할 |
|---|---|---|
| Frontend | React 19, TypeScript, Vite | SPA. 별도 라우터 없이 화면 상태(state)로 화면을 전환합니다. |
| Backend | Node.js, Express | `POST /api/mentor` 하나로 멘토 턴 생성 API를 제공하는 경량 서버입니다. |
| AI | OpenAI API (`gpt-4o-mini`), `response_format: json_object` | 자유 텍스트가 아니라 정해진 JSON 스키마(질문/최종 턴, 선택지, 내부 상태)로 응답을 강제합니다. |
| Database | SQLite, `better-sqlite3` | 서버 로컬 파일(`BE/data/mentor.sqlite`) 기반으로 대화 턴을 기록합니다. |
| 연동 | Vite Dev Server Proxy | 개발 환경에서 FE(`5173`)의 `/api/*` 요청을 BE(`8787`)로 프록시해 CORS 설정 없이 연동합니다. |
| Styling | CSS (커스텀, 프레임워크 미사용) | 모바일 앱 화면을 흉내 낸 `phone` 프레임과 초록/teal 테마로 구성됩니다. |

## 실행 방법

FE와 BE를 각각 별도 터미널에서 띄워야 전체 기능(멘토 채팅 포함)이 동작합니다.

```bash
# 0) 저장소 클론
git clone https://github.com/wonjunsea/cheongsanyusu.git
cd cheongsanyusu

# 1) BE 실행 (터미널 1)
cd BE
npm install
cp .env.example .env      # 발급받은 OPENAI_API_KEY를 채워넣기
npm run dev                # http://localhost:8787

# 2) FE 실행 (터미널 2, 프로젝트 루트에서)
cd FE
npm install
npm run dev                # http://localhost:5173 — /api/* 요청은 자동으로 BE(8787)로 프록시됨
```

## 확인 방법

1. 브라우저에서 `http://localhost:5173` 접속 → 종목 리스트 화면이 보이면 정상입니다.
2. "레버리지 · 인버스 · 급등주" 목록에서 아무 종목이나 클릭 → 케이스별 후킹 문구가 뜨는 전체 화면이 나오는지 확인합니다.
3. "확인하러 가기" 클릭 → 금융 멘토 채팅방 진입, 잠시 후 객관식 질문 1이 뜨는지 확인합니다. (뜨지 않으면 BE 터미널 로그와 `OPENAI_API_KEY` 설정을 확인하세요.)
4. 선택지를 눌러 질문 2 → 질문 3 → 최종 요약/설명까지 이어지는지 확인합니다.
5. 최종 화면의 행동 선택지를 누르면 케이스에 맞는 체험 화면(호가창 게임 / 음의 복리 시뮬레이션 / ETF 준비 중 화면)으로 이동하는지 확인합니다.
6. `BE/health` 엔드포인트(`http://localhost:8787/health`)가 `{"ok":true}`를 반환하면 서버가 정상 기동된 것입니다.
