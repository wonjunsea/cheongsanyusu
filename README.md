# 2026 TECH4GOOD HACKATHON 

## 팀명: 연9대상
## 서비스명: 청산유수

## 서비스 소개

**청산유수**는 투자 직전, 빚투·영끌 청년들의 계좌 청산을 막고 고위험 금융 상품의 위험성을 인지시키는 **30초 교육 솔루션**입니다.

청년 투자자가 레버리지, 인버스, 급등주 등 고위험 상품에 진입하기 전 짧은 시뮬레이션과 게임을 통해 상품의 위험성을 직접 체험할 수 있도록 설계했습니다.

복잡한 금융 교육을 길게 듣는 방식이 아니라, 실제 투자 의사결정 직전에 초보자도 쉽게 이해할 수 있는 체험형 콘텐츠를 제공하여 무리한 레버리지 진입을 줄이고 청년 투자자를 보호하는 것을 목표로 합니다.

---

## 해결하고자 하는 문제

| 문제 | 설명 |
|---|---|
| 청년 빚투·영끌 증가 | 소득이 불안정한 청년층이 위험을 충분히 이해하지 못한 채 고위험 상품에 진입하는 문제가 증가하고 있습니다. |
| 고위험 상품 이해 부족 | 레버리지, 인버스, 급등주 등은 손실 구조가 복잡하지만 투자자는 수익 가능성에만 집중하기 쉽습니다. |
| 기존 교육의 낮은 접근성 | 기존 금융 교육은 수동적이고 길어 실제 투자 직전의 사용자를 붙잡기 어렵습니다. |
| 투자 직전 개입 부족 | 위험한 선택이 일어나는 바로 그 순간에 위험을 체감시키는 장치가 부족합니다. |

## 주요 기능

| 기능 | 설명 |
|---|---|
| AI 금융 멘토 에이전트 | OpenAI API 기반 금융 멘토가 사용자의 투자 판단과 위험 인식을 점검합니다. |

## AI·데이터 활용 방식 아키텍처

### 전체 흐름

```
고위험 상품 클릭 (레버리지 / 인버스 / 급등주)
     │
     ▼
[FE] 진입 훅 화면 — 종목별 케이스에 맞는 한 줄 후킹 문구
     │  ("2배 ETF면 더 벌어야 하는 거 아닌가요?" 등)
     ▼
[FE] 금융 멘토 채팅 화면 진입 → BE에 턴 요청
     │  POST /api/mentor { system, caseType, user_id, answeredTurns }
     ▼
[BE] OpenAI(gpt-4o-mini) 호출, JSON 모드로 응답 형식 강제
     │  system 프롬프트 = 공통 롤플레잉 규칙 + 케이스별 착각 패턴/상태 라벨
     ▼
[BE] 구조화된 턴 JSON 반환 + 사용자 답변을 SQLite에 저장
     │  { turn: {...}, inferred_state: {...} }
     ▼
[FE] 객관식 선택지 렌더링 → 사용자 선택 → 다음 턴 요청 (질문 1~3 반복)
     │
     ▼
[BE] 4번째 턴에서 최종 요약 + 개념 교정 + 행동 선택지 생성
     │
     ▼
[FE] 케이스에 매칭된 체험 화면(게임/시뮬레이션)으로 이동
```

### AI 활용 방식

- **모델**: OpenAI `gpt-4o-mini`, `response_format: json_object`로 자유 텍스트가 아닌 고정 스키마 출력을 강제
- **턴 구조**: 정확히 4턴(객관식 질문 3회 + 최종 턴 1회)으로 제한 — LLM이 임의로 대화를 늘리거나 줄이지 않도록 서버가 매 요청마다 턴 인덱스를 계산해 지시
- **내부 상태 추론**: 매 응답마다 LLM이 `mental_model`(개념 이해 방식) · `risk_belief`(위험 인식) · `market_frame`(시장 해석 프레임) 등을 함께 추론해 반환 → 다음 질문의 축(5W1H 중 what/why/when/how/where/who)을 결정하는 데 사용
- **케이스별 프롬프트**: 종목마다 고정된 케이스(급등주 FOMO / 음의 복리 / 인버스 ETF)에 맞는 착각 패턴과 상태 라벨을 프롬프트에 주입해, 같은 스키마 안에서 서로 다른 금융 개념을 다룸

### 데이터 활용 방식

- **저장소**: SQLite(`better-sqlite3`), 서버(BE) 로컬 파일로 관리
- **스키마**: `user_id` + `case_type` 기준으로 각 턴의 질문/선택 답변을 순서대로 기록
- **사용자 식별**: 로그인 없이 `localStorage`에 저장된 랜덤 UUID로 세션을 구분하는 프로토타입 방식
- **활용 목적**: 현재는 대화 로그 적재까지만 구현되어 있고, 이 데이터를 이용한 사용자 성향 집계·리포트는 후속 확장 지점으로 남겨둠
